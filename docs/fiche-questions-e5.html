01 — Questions TiBillet / Lespass

❓ "C'est quoi Docker ?"

Outil de conteneurisation : chaque service (Fedow, Lespass, PostgreSQL) tourne dans un conteneur isolé avec ses propres dépendances. Avantage : reproductible sur n'importe quelle machine sans conflit de versions.

❓ "C'est quoi Django ?"

Framework web Python qui suit le pattern MVC (Modèle-Vue-Template). TiBillet l'utilise pour la base de données (models), la logique métier (views) et l'affichage HTML (templates).

❓ "Pourquoi Fernet ?"

Chiffrement symétrique. TiBillet l'utilise pour sécuriser les échanges entre Fedow et Lespass — les deux .env partagent la même clé. Si elles diffèrent, les moteurs ne communiquent plus.

❓ "C'est quoi un BooleanField ?"

Champ Django qui stocke True/False en base. Pour show_time et show_price_note, il permet à l'admin d'activer/désactiver un affichage sans modifier le code.

❓ "C'est quoi une migration Django ?"

Quand on ajoute un champ au modèle, Django doit mettre à jour la BDD. makemigrations génère le fichier de migration, flush.sh l'applique ici en réinitialisant aussi les données.

❓ "C'est quoi un tenant Django ?"

Isolation multi-tenant : public (utilisateurs), lespass (billetterie), meta (agrégateur). Permet à une instance TiBillet de gérer plusieurs orgas sur la même BDD.

❓ "Vous avez contribué au code officiel de TiBillet ?"

Non — j'ai travaillé sur le fork de Komidi. Ma mission était d'adapter TiBillet aux besoins de l'association. Une contribution upstream aurait nécessité des tests approfondis et une validation par Code Commun.

❓ "Comment avez-vous diagnostiqué le Bug VPS ?"

Les logs Django montraient une exception liée au résolveur de tenant. En inspectant le modèle Client via le shell Django, j'ai trouvé que la categorie du schema_name était incorrecte. Correction via manage.py shell.

❓ "C'est quoi AGPL-3.0 ?"

Licence copyleft forte — toute modification doit être rendue publique sous la même licence, même pour un service en ligne. C'est ce qui garantit que TiBillet reste libre même si des orgas l'adaptent.

❓ "Les mentions légales, c'est du développement ?"

C'est de la production de contenu avec contrainte légale forte — LCEN et RGPD. La mise en ligne d'un service implique ces obligations. C'est une réalisation à part entière → compétence C3.

❓ "Pourquoi VSCodium et pas VS Code ?"

VSCodium est la version open source de VS Code sans télémétrie Microsoft — cohérent avec la philosophie open source et coopérative de TiBillet et de Komidi.

❓ "C'est quoi Fedow / Lespass / LaBoutik ?"

3 moteurs TiBillet. Fedow = fédération et transactions. Lespass = billetterie et adhésions. LaBoutik = caisse cashless (non utilisé chez Komidi).

02 — GLPI — Gestion du parc & tickets · C1 + C2 · ⚠️ Beaucoup de questions selon les jurys

❓ "C'est quoi GLPI ?"

Logiciel open source de gestion du parc informatique et des tickets d'incidents. Il suit les bonnes pratiques ITIL — inventaire des équipements, suivi des incidents, gestion des demandes de service.

❓ "C'est quoi un parc informatique ?"

L'ensemble des équipements matériels et logiciels d'une organisation — ordinateurs, imprimantes, serveurs, licences. GLPI permet de les inventorier, les localiser et suivre leur état.

❓ "C'est quoi un ticket dans GLPI ?"

Formulaire de déclaration d'un problème ou d'une demande. Il contient le type (incident ou demande), la priorité, l'utilisateur concerné, la description du problème et l'historique des actions de traitement.

❓ "Différence entre incident et demande de service ?"

Un incident est une interruption non planifiée d'un service (ex : ordinateur qui ne démarre plus). Une demande est une sollicitation standard prévisible (ex : installation d'un logiciel, création d'un compte).

❓ "C'est quoi ITIL ?"

Référentiel de bonnes pratiques pour la gestion des services informatiques. Il structure notamment la distinction incident/demande, le cycle de vie des tickets, et les niveaux de priorité. GLPI s'appuie sur ces pratiques.

❓ "Vous avez utilisé GLPI comment ?"

En TD au lycée — simulation de la gestion d'un parc : création d'actifs (ordinateurs, imprimantes), ouverture de tickets d'incidents, traitement et affectation à des techniciens, suivi de l'état jusqu'à la résolution.

❓ "Comment on crée un actif dans GLPI ?"

Dans le menu Parc → Ordinateurs (ou autre type d'équipement) → Ajouter. On renseigne le nom, le type, le lieu, le statut, l'utilisateur affecté et les caractéristiques matérielles.

❓ "Comment on traite un ticket dans GLPI ?"

On ouvre le ticket, on l'affecte à un technicien, on ajoute des suivis (actions effectuées), on change le statut (nouveau → en cours → résolu → clôturé) et on documente la solution appliquée.

03 — PharmaPays++ — Application Java desktop · C1 + C4 + C5 · Java · JavaFX · SQLite · Maven

❓ "C'est quoi PharmaPays++ ?"

Application desktop Java développée en équipe de 4 — permet de gérer et consulter un catalogue de médicaments pour une pharmacie fictive. Interface JavaFX, base de données SQLite déployée sur machine virtuelle.

❓ "C'est quoi un MCD / MLD ?"

MCD (Modèle Conceptuel) = représentation abstraite des entités et de leurs relations. MLD (Modèle Logique) = traduction en tables pour la base relationnelle. J'ai conçu le schéma entité-relation du projet.

❓ "Pourquoi SQLite et pas MySQL ?"

SQLite est une base embarquée, sans serveur dédié — adaptée pour une application desktop standalone. MySQL nécessiterait un serveur séparé, ce qui est inutile pour ce type de projet.

❓ "C'est quoi JavaFX ?"

Bibliothèque Java pour créer des interfaces graphiques desktop. Elle remplace Swing pour les projets modernes — fenêtres, boutons, tableaux — avec une syntaxe plus propre et un meilleur rendu visuel.

❓ "C'est quoi Maven ?"

Outil de gestion de projet Java — gère les dépendances (bibliothèques externes), la compilation et le packaging de l'application. C'est l'équivalent de npm pour Java.

❓ "Comment vous avez organisé le travail en équipe ?"

Répartition claire des modules dès le départ, après validation commune du modèle BDD par toute l'équipe. Mon rôle : conception du schéma entité-relation, création et déploiement de la base SQLite sur la VM.

04 — Tetris — Projet personnel Python · C6 · Python · Pygame · POO

❓ "Pourquoi vous avez fait un Tetris ?"

C'était mon premier projet personnel en dehors du cursus — Python était mon tout premier langage. Je voulais apprendre par la pratique sur un cas concret et motivant plutôt que par des exercices académiques.

❓ "C'est quoi Pygame ?"

Bibliothèque Python pour le développement de jeux 2D — gère l'affichage graphique, les événements clavier, les animations et la boucle de jeu principale.

❓ "C'est quoi la POO ?"

Programmation Orientée Objet — organiser le code en classes avec des attributs et des méthodes. Pour Tetris, j'ai séparé la logique du jeu (Tetris.py), l'affichage (Graphics.py) et le menu (menu.py).

❓ "Comment vous avez géré les collisions ?"

Avec une matrice de collision — à chaque déplacement de pièce, je vérifie si les cases cibles sont libres et dans les limites de la grille avant d'autoriser le mouvement. Résolu après recherches personnelles.

❓ "Comment vous avez géré les rotations ?"

Chaque pièce est représentée par une matrice. La rotation applique une transformation mathématique sur cette matrice — puis on vérifie les collisions avant de valider la nouvelle position.

❓ "Qu'est-ce que ça vous a apporté ?"

Ce projet a déclenché mon intérêt pour le développement logiciel. Il m'a montré que je pouvais construire quelque chose de complet en autonomie, et m'a motivé à apprendre Java en 2ème année pour des projets plus structurés.
